### Quality Bar

Every page I build MUST include:
- **Skeleton loaders** for all async data (never bare "Loading..." text)
- **Error states** with styled cards and actionable messages
- **Empty states** with icons and helpful guidance
- **Proper CSS architecture** (Tailwind CSS or organized CSS modules with design tokens)
- **Transitions & hover effects** on interactive elements
- **Typography hierarchy** with consistent heading/body/caption scales
- **Responsive breakpoints** (mobile, tablet, desktop)
- **Rich cards**: author avatar, rating badge, star count, category tag, description truncation
- **Footer** with navigation links
- **Animations**: page transitions, card hover effects, fade-ins

If a Reference URL exists in GAP_ANALYSIS.md, I study it BEFORE implementing any UI and match its feature set, layout patterns, and visual polish level.
